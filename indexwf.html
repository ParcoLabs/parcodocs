Hybrid
▾
04
Property Details Entry
Tokenizer completes structured property intake form: address, type (SFR/STR/multifamily), estimated value, monthly gross income, existing liens, entity structure preference. Platform validates each field with inline error states. Progress tracker shows completeness %.
Human Input React form UI Zod validation
⚙ Human Required
05
Document Upload & R2 Storage
Tokenizer uploads required documents: deed, rent roll (last 12 months), income/expense statement, entity documents, existing appraisal (if available). Each file is uploaded to Cloudflare R2 via signed URL. BullMQ queues each doc for AI extraction immediately on upload.
Hybrid Cloudflare R2 BullMQ job queue Redis
↗ Instant processing queue
06
AI Document Extraction (Claude API)
BullMQ worker pulls each document from R2 and sends to Claude API with a structured extraction prompt. Claude extracts 40+ fields per document: property address, legal description, APN, purchase price, lien amounts, gross income, NOI, occupancy %, entity name/type, ownership %. Extracted fields stored as structured JSON in PostgreSQL.
Automated Claude API (document extraction prompt) BullMQ worker Prisma / PostgreSQL
↗ 40 fields in <30 sec ⟳ vs $500–$2K paralegal work
07
Field Cross-Reference & Discrepancy Detection
System automatically compares extracted fields against tokenizer's stated values from the intake form. Any mismatch exceeding threshold (e.g., stated income $5K/mo vs extracted $3.2K/mo) is flagged as a risk item. Risk flags stored with severity levels (INFO / WARNING / BLOCKER). Tokenizer UI shows red/amber indicators on mismatched fields.
Automated Discrepancy engine (custom logic) Claude API (semantic comparison)
↗ Catches errors before review
PHASE 02 Parco Intelligence — AI Analysis & Deal Score
Fully Automated
▾
08
Independent AVM Valuation
Claude API runs an automated market valuation using extracted property data + comparable sales data from integrated market feeds. Computes estimated market value and delta from tokenizer's stated value. Delta feeds directly into Deal Score confidence. 5% gap = noted; 15%+ gap = BLOCKER flag requiring human review before approval.
Automated Claude API (valuation prompt) Market data feeds
↗ Replaces $500–$2K appraisal review
09
Pathway Confidence Scoring (Final)
Claude API runs the full pathway eligibility model with all extracted and verified data. Outputs final 0–100 confidence scores for Series LLC, Reg D 506(c), Reg D 506(b), and Reg A+. Primary pathway recommended with specific blockers listed for each alternative. Series LLC typically scores 85–95 for target properties.
Automated Claude API (pathway eligibility model)
↗ 97% match rate vs counsel decision
10
Risk Flag Aggregation & Prioritization
All risk flags from extraction, cross-reference, and valuation steps are aggregated and prioritized. Claude API categorizes each flag: BLOCKER (must resolve before approval), WARNING (requires reviewer acknowledgment), INFO (noted for record). Generates a ranked list with recommended resolution for each blocker.
Automated Claude API (risk classification) PostgreSQL (flag store)
11
Deal Score Report Generated
System compiles the one-page Deal Score: pathway confidence bars, document completeness %, risk flag summary, valuation delta, recommended pathway with rationale, and specific blockers. Report generated as structured JSON and rendered in the Admin Review UI. This is what the human reviewer sees — not raw documents.
Automated Claude API (report generation) React (Deal Score UI)
⟳ Replaces $10K–$20K legal review prep
12
Offering Packet First Draft
Claude API generates a draft Offering Packet: property summary, financial overview, token structure summary, key terms, risk disclosures template, and investor FAQ. Draft stored in R2. Marked as DRAFT — not shared with investors until admin approves final version after human review.
Automated (draft only) Claude API (offering packet prompt) Cloudflare R2
↗ Draft in minutes, not days ⟳ vs $5K–$10K drafting cost
